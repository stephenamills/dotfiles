# Excel workbook study-and-build manual generation

Create a standalone Markdown study-and-build manual from the deterministic workbook snapshot and any corresponding transcripts. Treat workbook formulas, cells, structures, formats, and relationships as authoritative for implementation; use transcripts for purpose, interpretation, workflow, and instructional intent. Do not browse or invent workbook content.

Teach both how the workbook works and how to reconstruct its intentional design from a blank workbook in current Microsoft 365 Excel for Mac. Cover workbook and sheet architecture, inputs, outputs, column/range roles, formula archetypes, reference behavior, dependencies, styles, validation, conditional formatting, operating workflow, build order, verification checkpoints, debugging, progressive practice, mastery questions with answers, and a glossary. Clearly identify extraction limits and unsupported legacy features.

Use exactly one H1 and coherent H2–H4 nesting. Include useful fenced D2 diagrams, never Mermaid. At minimum, include a D2 dependency diagram connecting source inputs, columns or ranges, formulas, intermediate calculations, checks, summaries, and decision outputs. Add build-order or debugging diagrams when they materially clarify the workbook.
